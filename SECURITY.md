# Security Policy

## Supported Versions

| SDK | Version | Supported |
|-----|---------|-----------|
| Swift SDK | 1.1.0 | Yes |
| Vapor SDK | 1.1.0 | Yes |
| JavaScript SDK | 1.1.0 | Yes |
| React Native SDK | 1.1.0 | Yes |
| Flutter SDK | 1.1.0 | Yes |
| Kotlin/Android SDK | 1.1.0 | Yes |

## Reporting a Vulnerability

If you discover a security vulnerability in any FeedbackKit SDK, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, please email us at **info@swiftly-developed.com** with:

- A description of the vulnerability
- Steps to reproduce the issue
- The affected SDK and version
- Any potential impact assessment

## Response Timeline

- We will acknowledge your report within **48 hours**
- We will provide an initial assessment within **5 business days**
- We will work with you to understand and resolve the issue promptly

## Disclosure Policy

We follow a coordinated disclosure process:

1. The vulnerability is reported privately
2. We confirm and assess the issue
3. We develop and test a fix
4. We release the fix and publish a security advisory
5. Public disclosure occurs after the fix is available

We appreciate your help in keeping FeedbackKit and its users safe.
